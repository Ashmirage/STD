# 电位器 Pot

- 模块文件：`driver/bsp/Pot.h`、`driver/bsp/Pot.c`
- 当前引脚：`PA5 = ADC1_IN5 = STM_ADC`
- 当前实现：`单通道 + 轮询读取`
- 当前已实测成功方案：`板载 RV1 -> 多功能端口 -> STM_ADC(PA5)`

## 这次怎么移植的

参考工程里的 `Pot` 模块本身只负责取数，真正依赖的是另外一个 `AD + DMA` 模块。

但当前 `F407ZGT6` 主工程里没有现成的 ADC 框架，所以这次没有继续照搬 `AD_Value[]` 那套写法，而是直接在 `Pot` 里做了一个最小可用版本：

- 初始化 `ADC1`
- 把 `PA5` 配成模拟输入
- 每次调用 `Pot_GetData()` 时启动一次转换
- 轮询等待转换完成后直接返回结果

这样更短，也更适合当前工程直接用。

另外，这次根据《探索者 V3 硬件参考手册》的 IO 分配重新换过脚：

- 之前用的 `PC5` 在板上对应 `RMII_RXD1`，会和板载以太网功能复用，不适合继续当默认 ADC 脚
- 现在改成 `PA5`，手册里标的是 `STM_ADC`

## 当前实测成功的用法

当前已经确认能正常工作的接法是：

1. 拔掉 `TPAD <-> STM_ADC` 的跳线帽
2. 在多功能端口上，把板载 `RV1` 的滑动端接到 `STM_ADC`
3. 保持当前代码不变，直接读取 `Pot_GetData()`

实测现象：

- 转动板载 `RV1`，串口打印的 `pot` 数值会跟着变化
- `Pot_GetVoltage()` 也会随之变化
- 不需要再改 ADC 通道

## 接线

### 1. 用板载 RV1

推荐直接按上面的“当前实测成功方案”来接。

### 2. 用外部三线电位器模块

- `AO` 接 `PA5`
- `VCC` 接 `3.3V`
- `GND` 接 `GND`

## 和板载模块的关系

这一块一定要注意，不然很容易测错：

- `PA5` 在手册里是 `STM_ADC`，同时也能通过跳线拿去做 `TPAD` 触摸按键检测
- 如果你板子上把 `STM_ADC` 和 `TPAD` 短接了，先把这个跳线帽拔掉，再测电位器
- 板载可调电阻 `RV1` 的滑动端是引到多功能端口上的，不是直接硬连到 `PA5`
- 你要用板载 `RV1` 时，应该把 `RV1` 的输出通过多功能端口接到 `STM_ADC(PA5)`
- 这样就能继续用当前代码，不需要改 ADC 通道
- `STM_DAC(PA4)` 也可以做模拟输入，但那一路当前和电机 `PA4` 冲突，不建议拿来接板载 `RV1`

也就是说：

- 如果你接的是外部电位器模块，就按现在的 `PA5` 用
- 如果你想用板载 `RV1`，就把 `TPAD` 跳线拔掉，再把 `RV1` 接到 `STM_ADC(PA5)`
- 只有当你想把 `RV1` 接到 `PA4` 时，才需要处理 `Motor` 模块的冲突

## 板载 RV1 怎么接

按当前工程的 `PA5` 版本，推荐这样接：

1. 拔掉 `TPAD <-> STM_ADC` 的跳线帽
2. 在多功能端口上，把 `RV1` 滑动端接到 `STM_ADC`
3. 保持当前代码不变，直接读取 `Pot_GetData()`

这样做的好处是：

- 不用改代码
- 不占用 `PA4`
- 不会和当前 `Motor` 模块直接打架
- 已经按这个方案测试通过

## 接口

```c
#include "Pot.h"

Pot_Init();

uint16_t raw = Pot_GetData();   // 0 ~ 4095
float voltage = Pot_GetVoltage();
```

## 工程里的接入

当前已经在 `Hardware_init()` 里调用：

```c
void Hardware_init(void)
{
    // 其他初始化...
    Pot_Init();
}
```

这样上电后 ADC 就已经准备好了，后面直接读即可。

## 调试方法

最简单的测试方式：

```c
Send_printf("pot=%d, v=%.2f\r\n", Pot_GetData(), Pot_GetVoltage());
```

转动电位器时，应当看到：

- `pot` 在 `0~4095` 之间变化
- `v` 大致在 `0.00~3.30V` 之间变化

如果你现在用的是板载 `RV1`，重点看这两点：

- 不转动时，数值应当基本稳定，不应该像 `TPAD` 那样一碰就明显跳变
- 转动 `RV1` 时，数值变化应当连续，不是随机乱跳

## 常见问题

### 1. 数值一直不对

优先检查：

- `TPAD <-> STM_ADC` 跳线帽是否已经拔掉
- 多功能端口上，`RV1` 是否真的接到了 `STM_ADC`
- 当前测的是板载 `RV1`，还是外接电位器

### 2. 一摸板子数值就乱跳

大概率还是：

- `TPAD` 没断开
- 或者 `STM_ADC` 这一路还和别的板载功能串在一起

### 3. 想走 `PA4`

不推荐。因为：

- `PA4` 当前已经被 `Motor` 模块占用
- 你如果把电位器再接到 `PA4`，就会和电机方向控制冲突

## 说明

- 这是 12 位 ADC，所以满量程是 `4095`。
- 当前按 `3.3V` 参考电压换算电压值。
- 采样时间用了较长的 `144Cycles`，对电位器这类模拟量更稳一些。
- 这次没有上 DMA，也没有做多通道扫描，目的就是先保证简单直接、抄上去能用。
- 当前版本已经可以直接配合板载 `RV1` 使用，前提是按上面的跳线和多功能端口方式接好。

## 如果要改引脚

改文件：`driver/bsp/Pot.c`

主要改这些宏：

- `POT_GPIO_PORT`
- `POT_GPIO_PIN`
- `POT_GPIO_CLK`
- `POT_ADC`
- `POT_ADC_CLK`
- `POT_ADC_CHANNEL`

如果你后面还要接别的模拟量，也建议继续沿用这种写法，先单通道跑通，再考虑统一成多通道 ADC 模块。
